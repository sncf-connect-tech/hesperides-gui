Production role

Overview

The production role is a permission granted to some users to edit and have access to unobfuscated password on some production platforms.

What are production platforms?

Platforms can be tagged as production in 3 places :

Where ? How ? Result in logic representation
Create platform
Create platform from
Logic representation

What is restricted?

When a user without production role access a production platform:

  • they won't be able to save any changes made to the platform
  • they won't be able to see the value of properties with @password annotations: ******** will be displayed instead

Who has production role?

  • users belonging to the global production group, configured through the configuration environment variable $LDAP_PROD_GROUP_DN
  • users who belong to the directory groups configured for delegating those permissions per-application, the so-called ACLs

Editing ACLs

ACLs can be edited easily through the form at the top of every application page, where users already having the production role can delegate those permissions to additional directory groups. A user ACLs are summarized on the profile page.
